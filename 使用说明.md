# 盘古AI写作系统 · 使用说明

> 版本：V7.5
> 更新：2026-06-08

---

## 核心脚本

| 脚本 | 用途 | 什么时候用 |
|------|------|-----------|
| **`pangu_pipeline.py`** | 一键流水线：方向→参考书→提示词→DeepSeek→归档 | 每次开始新小说 |
| `pangu_optimized.py` | 项目管理 + 续写章节 | 已有项目继续写后续章节 |

---

## 一、从零开始一篇小说

只需要一句话：

```bash
python pangu_pipeline.py "重生回到高考前一天，用未来十年记忆逆袭" --genre 都市 --platform qimao
```

### 系统自动完成的五步：

```
① 搜参考书 → 在753本参考书中找同题材，分析句长/对话率/钩子类型
② 出提示词 → 注入平台风格 + 参考书风格摘要
③ DeepSeek → 生成完整大纲
④ DeepSeek → 生成第1章正文
⑤ 归档     → 创建 projects/书名/ 目录，保存大纲、正文、state.json
```

### 参数说明：

```
pangu_pipeline.py "概念" 
    --genre 都市|玄幻|科幻|历史|悬疑|体育|二次元     (默认:玄幻)
    --platform qimao|fanqie|qidian|jinjiang          (默认:qimao)
    --ref-count 5                                     (参考书数量，默认5)
    --skip-outline                                    (只生成第1章)
    --skip-chapter1                                   (只生成大纲)
```

### 平台风格参考：

| 平台 | 一句话 |
|------|--------|
| qimao | 七猫——快节奏，短句多对话，情绪浓烈不克制，章章有钩子 |
| fanqie | 番茄——大白话，开篇300字出冲突，主角杀伐果断 |
| qidian | 起点——允许中长句，世界观纵深感，人物成长弧光 |
| jinjiang | 晋江——细腻五感，对话有潜台词，情绪渗透型不直给 |

## 二、已有项目续写

```bash
python pangu_optimized.py
```

进入交互菜单：
- `[1]` 快速创建项目（5种模板可选）
- `[2]` 查看已有项目列表，选择项目进入
- 项目内操作：
  - `[1]` 快速写新章节（一句话任务 → 自动生成）
  - `[2]` 批量生成（一次生成多章）
  - `[3]` 查看/打开大纲
  - `[4]` 打开最新章节

JSON文件保留作为向后兼容的降级方案。

## 五、五车间流水线（高级模式）

完整版逐章走 W0→W1→W2→W3→W4，每步记录到数据库：

```
W0: 主旨锚定 → 锁定核心主旨，防跑偏
W1: 设定预处理 → 提取本章热库（500字）
W2: 正文初稿 → 干净剧情骨架
W3: 逻辑质检 → 检查OOC/伏笔/规则
W4: 文笔精修 → 添加氛围和质感
```

在 `pangu_optimized.py` 中启用车间模式（开发中）。

## 七、配置

编辑 `.env` 文件：

```
DEEPSEEK_API_KEY=你的密钥
AI_MODEL=deepseek-chat
```

## 八、文件结构

```
盘古AI/
├── pangu_pipeline.py          ← 一键流水线 ★ 主力
├── pangu_optimized.py         ← 项目管理 + 续写
├── pangu_plus.py              ← 原版（功能最全）
├── knowledge/
│   ├── novel_reference.db     ← 统一数据库（450MB）
│   ├── unified_db_manager.py  ← 统一DB管理器
│   ├── workshop_db_manager.py ← 车间DB管理器
│   ├── chapter_analyzer.py    ← 章节风格分析器
│   ├── reference_prompt.py    ← 参考提示词生成器
│   ├── import_unified.py      ← 统一导入工具
│   └── batch_analyze.py       ← 批量分析法
├── workshops/                 ← 五车间 System Prompts
├── backend/                   ← Flask API + RAG + 可观测性
├── modes/                     ← 12种创作模式（JSON降级）
└── projects/                  ← 所有小说项目
```
