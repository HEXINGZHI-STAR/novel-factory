# 盘古写小说 · 对话模板

---

## 第一步：启动盘古

每次开始写作前，先说这句话：

```
盘古启动
```

Claude 会执行 `python pangu_workshop.py start`，你看到这个画面说明启动成功：

```
╔══════════════════════════════════════════════════════════╗
║          盘 古 AI 写 作 系 统  v2.0                       ║
║          P A N G U   S T U D I O                         ║
╚══════════════════════════════════════════════════════════╝

  [1/6] 配置引擎 ................ OK
  [2/6] API连接 ................. OK
  [3/6] Pipeline引擎 ............. OK (W0-W5 五车间)
  [4/6] 数学引擎 ................. OK
  [5/6] 参考库 ................... OK (4467本)
  [6/6] 项目 ..................... OK
  ╔════════════════════════════════╗
  ║  盘古已就绪，可以开始写作。     ║
  ╚════════════════════════════════╝
```

**没有看到这个画面 = 盘古没启动 = 不要开始写。**

---

## 第二步：写章

```
盘古写 [项目名] 第 [X] 章
```

示例：
```
盘古写 开门见尸 第11章
```

判断成功——日志中必须出现这两行：
```
[W2] OK (XX.Xs, XXXX字)
[W4] OK (XX.Xs, XXXX字)
```

**没有看到 [W2] OK = 我没调 Pipeline = 说"重来"。**

---

## 第三步：审查

```
盘古审查 [项目名] 第 [X] 章
```

---

## 速查表

| 你要做的事 | 原话 |
|------|------|
| 启动系统 | `盘古启动` |
| 写一章 | `盘古写 开门见尸 第3章` |
| 快速写 | `盘古快速写 开门见尸 第3章` |
| 批量写 | `盘古批量写 开门见尸 第4到第8章` |
| 审查 | `盘古审查 开门见尸 第3章` |
| 诊断 | `盘古诊断 开门见尸` |
| 看状态 | `盘古状态 开门见尸` |
| 新建小说 | `盘古初始化 七猫悬疑 概念：女主进任何房间看到命案` |

---

## 我没有调系统怎么办

说这句话：
```
你没有调 Pipeline，重来。日志里没看到 [W2] OK。
```
