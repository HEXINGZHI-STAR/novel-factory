# 句式结构的数学模型（真人网文写作版 v3）

> **核心思想**：把写作从"感觉"变成"可控的参数"。每一段文字都是一组句子长度、类型、标点的联合分布。

---

## 一、核心变量定义（Notation）

| 符号 | 含义 | 取值范围 | 真人目标值（通用） |
|------|------|---------|-------------------|
| **n** | 段落内句子总数 | n ≥ 3 | 视段落而定 |
| **L_i** | 第 i 句的字数（含标点） | 1 ≤ L_i ≤ 200 | |
| **μ_L** | 段落平均句长 = (1/n) Σ L_i | 15 ≤ μ_L ≤ 50 | **25-35** |
| **σ_L** | 句长标准差 = √[ (1/n) Σ (L_i - μ_L)² ] | 5 ≤ σ_L ≤ 30 | **10-15** |
| **CV_L** | 句长变异系数 = σ_L / μ_L | 0.2 ≤ CV_L ≤ 0.8 | **0.35-0.5** |
| **p_short** | 短句率 = count(L_i ≤ 10) / n | 0 ≤ p_short ≤ 0.5 | **≤ 0.20** |
| **p_mid** | 中句率 = count(11 ≤ L_i ≤ 30) / n | 0.15 ≤ p_mid ≤ 0.7 | **0.35-0.45** |
| **p_long** | 长句率 = count(L_i ≥ 31) / n | 0.1 ≤ p_long ≤ 0.7 | **0.40-0.50** |
| **p_describe** | 描写句占比 | 0.15 ≤ p_describe ≤ 0.45 | **0.25** |
| **p_narrate** | 叙述/背景推进句占比 | 0.15 ≤ p_narrate ≤ 0.45 | **0.25** |
| **p_action** | 动作句占比 | 0.05 ≤ p_action ≤ 0.35 | **0.15-0.20** |
| **p_dialogue** | 对话句占比 | 0 ≤ p_dialogue ≤ 0.35 | **0.05-0.15** |
| **p_reveal** | 极短揭示句占比（1-10字独立） | 0.05 ≤ p_reveal ≤ 0.2 | **0.10** |
| **H_punct** | 标点熵 = - Σ p_k · log2(p_k)，k∈{。？！，：；—} | 0.8 ≤ H_punct ≤ 2.0 | **1.2-1.5** |
| **r_q** | 问号率 = count(？) / n | 0.05 ≤ r_q ≤ 0.3 | 题材相关 |
| **r_e** | 叹号率 = count(！) / n | 0 ≤ r_e ≤ 0.5 | 题材相关 |

**关键恒等式**：p_short + p_mid + p_long = 1，p_describe + p_narrate + p_action + p_dialogue ≤ 1（有重叠和其他类型）

## 二、题材→参数映射表（Genre Parameter Vectors）

每个题材对应一个7维参数向量 **θ_genre = [μ_L, p_long, p_describe, p_narrate, p_action, r_q, r_e]**

| 题材 | μ_L目标 | p_long目标 | p_describe | p_narrate | p_action | r_q | r_e |
|------|--------|-----------|------------|-----------|----------|-----|-----|
| 玄幻/仙侠 | **30** | 0.40 | 0.22 | 0.25 | 0.16 | 0.10 | 0.15 |
| 历史/权谋 | **35** | **0.53** | **0.30** | 0.28 | 0.05 | 0.15 | 0.05 |
| 悬疑/无限流 | **50** | **0.57** | **0.41** | 0.10 | 0.05 | **0.26** | 0.16 |
| 军事 | **48** | **0.61** | 0.36 | 0.28 | 0.08 | 0.18 | **0.49** |
| 体育/爽文 | **22** | 0.28 | 0.15 | 0.15 | **0.23** | 0.08 | 0.12 |
| 西方奇幻 | **32** | 0.43 | **0.37** | 0.17 | 0.08 | 0.11 | 0.08 |
| 科幻/都市科技 | **28** | 0.42 | 0.20 | **0.34** | 0.10 | 0.15 | 0.08 |

### 2.1 参数向量的使用方法（实际写作时）

对于选定题材，生成一段文字时必须满足：
```
目标函数 minimize | μ_L(你写的) - μ_L(target) |
约束条件:
    | p_long(你写的) - p_long(target) | < 0.15
    | p_describe(你写的) - p_describe(target) | < 0.10
    CV_L ≥ 0.30           （必须有句长变化，不许全是一样长的句子）
    max(L_i) / min(L_i) ≥ 5   （最长句 / 最短句 ≥ 5，制造节奏冲击）
```

### 2.2 句长的推荐概率分布（Probability Mass Function）

真人网文的句长分布**不是均匀分布**，而是"中等句最多，长短句点缀"：

```
L = 1-10字:   P(L) ≈ 0.20     ← 揭示句、冲击句
L = 11-20字:  P(L) ≈ 0.25     ← 紧凑动作/推进
L = 21-30字:  P(L) ≈ 0.25     ← 标准叙述
L = 31-50字:  P(L) ≈ 0.20     ← 细节描写、复合句
L = 51-80字:  P(L) ≈ 0.08     ← 长铺垫、环境渲染
L ≥ 81字:     P(L) ≈ 0.02     ← 极少，仅在高潮或史诗感时使用
```

⚠️ **AI常见偏差**：AI 倾向于写 **L ∈ [5, 15]** 的句子，概率分布严重集中在 p(L=5-15) ≈ 0.8，CV_L ≈ 0.1（几乎不变），max/min ≈ 2-3。这就是"AI味"的数学来源。

## 三、句间关联函数（Inter-sentence Coherence Function）

句子不是孤立的。第 i 句和第 i+1 句的关系必须服从以下转移概率矩阵 **T**（基于16部网文统计的平均值）：

### 3.1 句间类型转移矩阵 T[type_i → type_{i+1}]

行=当前句类型，列=下一句类型，值=转移概率 P(下一句类型 | 当前句类型)

| 当前句 \ 下一句 | 描写 | 叙述/推进 | 动作 | 对话 | 极短揭示 |
|----------------|------|----------|------|------|---------|
| **描写**       | 0.32 | 0.28     | 0.15 | 0.10 | 0.15    |
| **叙述/推进**   | 0.25 | **0.35** | 0.18 | 0.12 | 0.10    |
| **动作**       | 0.18 | 0.22     | 0.20 | **0.28** | 0.12    |
| **对话**       | 0.15 | 0.20     | 0.25 | 0.30 | 0.10    |
| **极短揭示**   | **0.40** | 0.20 | 0.15 | 0.10 | 0.15 |

**解读**：
- 描写句之后最可能继续描写 (0.32) 或转为叙述 (0.28) —— 场景铺开后，给信息
- 叙述之后最可能继续叙述 (0.35) —— 推进信息是惯性
- 动作之后最可能接对话 (0.28) —— 人物动了之后往往开口
- **极短揭示句之后最可能接描写 (0.40)** —— "一张脸。" 之后紧跟着渲染环境，这就是冲击感的来源

### 3.2 句长转移模式（Length Transition Pattern）

真人作者的句长序列不是随机的，而是有明显的**节奏模式**：

| 模式 | 句长序列 L_1, L_2, L_3... | 出现场景 |
|------|--------------------------|---------|
| **上升模式** | 15 → 25 → 40 → 60 | 紧张升级、发现真相、情绪积累 |
| **下降模式** | 50 → 30 → 15 → **5** | 从铺垫到冲击揭示（最高效的钩子模式） |
| **交替模式** | 40 → **8** → 35 → **12** → 28 | 悬念感、犹豫感、碎片化思考 |
| **稳定模式** | 25-35 字保持 | 平静叙述、对话推进 |
| **极值冲击** | 80 → 60 → 40 → 20 → **3** | 史诗感→高潮→最后一句的震撼 |

**下降模式（最推荐用于章节结尾）**：L_i+1 = α · L_i，其中 0.4 ≤ α ≤ 0.7，直到 L ≤ 10。
例：55 → 30 → 18 → **8**（章末的钩子句）

## 四、段落结构的数学约束

### 4.1 段落长度变异约束

对于一章内的 m 个段落，设 P_k 为第 k 段的字数：
```
必须满足: max(P_1, P_2, ..., P_m) / min(P_1, P_2, ..., P_m) ≥ 5
即: 最长段落的字数 ≥ 5 × 最短段落的字数
```
这保证了长短段交替的节奏感。AI 常见偏差是所有段落都在 80-150 字之间，CV_P < 0.2。

### 4.2 单段内的句式分布约束

对于单个段落（句子数 n ≥ 3）：
```
约束1: 不得出现 |L_i - L_{i+1}| ≤ 2 连续 ≥ 3次（不许连续3句一样长）
约束2: 段内至少有1个 L_i ≥ 30（每段至少一句像样的长句）
约束3: 段内至少有1个 L_i ≤ 10 或 段内有至少1次标点类型变化
约束4: 段内不得全是陈述句（句号），应有问号/叹号/破折号点缀
```

### 4.3 标点约束公式

```
目标: 0.5 ≤ r_period ≤ 0.85     句号率（陈述句主导，但不是100%）
       0 ≤ r_question ≤ 0.3    问号率（悬疑类可高）
       0 ≤ r_exclamation ≤ 0.5  叹号率（军事/玄幻可高，悬疑类低）
       r_question + r_exclamation ≤ 0.4  （不许通篇都是"！？"）

AI常见偏差: r_period ≈ 0.95, r_question ≈ 0.03, r_exclamation ≈ 0.02
           → 全是陈述句，完全没有情绪/疑问波动
```

## 五、反AI的句法禁令（硬约束）

以下模式在真人网文中出现概率 ≤ 0.01，但AI输出概率常常 ≥ 0.5。写作时必须硬性排除：

### 5.1 禁用的句式模板

| 禁用模式 | 数学表达 | 替代方案 |
|---------|---------|---------|
| "他感到/他心中/他暗道/他心里" + 情绪词 | count("他感到") / n ≤ 0 | 直接写内心内容，不加前缀 |
| "缓缓地/淡淡地/微微地/静静地/轻轻地" | count(副词堆叠) / 1000字 ≤ 2 | 用具体动作替代副词 |
| "不是……而是……"判断结构 | P("不是...而是") ≤ 0.001 | 直接陈述事实，不做对比判断 |
| 连续5句都是 L_i ≤ 12字的短句 | L_{i..i+4} ≤ 12 → 禁止 | 在第3或第4句插入长句 |
| 一段内同时"描写+心理+对话+动作"四种类型 | type_count(段内) ≥ 4 → 禁止 | 每段只承载1-2个信息通道 |
| "忽然/突然/猛然/骤然" | count(突然族) / 1000字 ≤ 3 | 用因果或事态演化替代"突然" |
| 连续3段都是 50字 ≤ P_k ≤ 150字的中段 | P_{k..k+2} ∈ [50,150] → 禁止 | 在中段之间插入极短段 |

### 5.2 词汇替换表（具体 > 抽象）

把AI最爱的"抽象词"换成"具体动作/细节"：

| AI写法（抽象） | 真人写法（具体） |
|--------------|----------------|
| 他很紧张 → | 他的喉结动了一下，没有说话。 |
| 她很高兴 → | 她把信折了三次，塞进内衣口袋。 |
| 气氛很压抑 → | 灯泡在头顶嗡嗡地响，照得每个人的脸都发黄。 |
| 他心里一惊 → | 他的手指在刀柄上停了半秒。 |
| 他恍然大悟 → | 他沉默了三秒，然后慢慢点了点头。 |

## 六、写作自检算法（Checklist Algorithm）

写完任意一段文字后，运行以下检查。**任何一项不达标都要重写该段。**

```
输入: 一段文字（拆分为n个句子 S_1..S_n，对应长度 L_1..L_n，类型 T_1..T_n）
输出: {pass: bool, issues: [string]}

算法 STEP-by-STEP:

1. 计算 μ_L = (1/n) Σ L_i
   if μ_L < 20 → FAIL "句子太短，平均句长不足20字"
   
2. 计算 p_long = count(L_i ≥ 31) / n
   if p_long < 0.25 → FAIL "长句太少，不到25%"
   
3. 计算 p_short = count(L_i ≤ 10) / n
   if p_short > 0.35 → FAIL "短句太多，超过35%"
   
4. 计算 CV_L = σ_L / μ_L
   if CV_L < 0.25 → FAIL "句长变化太小，像AI写的"
   
5. 检查连续短句: if 存在 i 使得 L_i ≤ 12 AND L_{i+1} ≤ 12 AND L_{i+2} ≤ 12
   → FAIL "连续3句都是短句"
   
6. 检查 max(L_i) / min(L_i)
   if 比值 < 4 → FAIL "最长句 / 最短句 < 4，没有节奏冲击"
   
7. 检查标点熵 H_punct
   if H_punct < 0.8 → FAIL "标点太单调，基本全是句号"
   
8. 检查禁用词: count(他感到/他心中/缓缓地/淡淡地/微微地/静静地/轻轻地/忽然/突然/骤然/不是...而是)
   if count > 0 → FAIL "包含AI痕迹词汇"
   
9. 检查句间转移合理性
   count(描写→描写) / n 不应超过0.35（不能一直纯描写）
   count(对话→对话) / n 不应超过0.30（对话之间要有动作或叙述间隔）

10. if 所有检查通过 → PASS
```

## 七、快速参考卡（写作时抬头就能看）

### 7.1 通用目标参数（任何题材都要遵守）
```
平均句长 μ_L ≥ 25字
长句率 p_long ≥ 0.40
短句率 p_short ≤ 0.25
句长变异系数 CV_L ≥ 0.30
最长/最短 ≥ 5
标点熵 H_punct ≥ 1.0
每1000字: "缓缓地/淡淡地/微微地" ≤ 2个
每1000字: "忽然/突然/猛然" ≤ 3个
绝对不出现: "他感到/他心中/他暗道/他心里" + 情绪形容词
```

### 7.2 改写操作手册（从AI味改到真人味）

```
操作1: 加长 + 细节注入
  原: 他睁开眼睛。                        (L=5, 动作)
  改: 他缓缓睁开眼，首先闻到一股淡淡的药草气息，然后才发现自己躺在一张简陋的石床上，浑身骨头仿佛散了架一般。  (L=48, 动作+描写)
  Δμ_L: +43, 把1句拆成信息密度更高的1句，或扩展成1句长短混合

操作2: 极短揭示 + 长句铺垫（下降模式）
  原: 房间中央站着一个人。                  (L=8, 叙述)
  改: 他瞳孔骤然收缩——窗外不是什么都没有，而是有什么东西贴在玻璃上，距离他的脸不到十厘米。(L=38, 描写+动作)
      一张脸。                                                           (L=3, 揭示)
  Δ: 38 → 3, 下降率 α = 0.08, 产生最大冲击

操作3: 对话嵌入叙述（不要每句对话换行）
  原: 老人说:"你醒了。"                    (L=10, 对话)
      少年点了点头。                        (L=6, 动作)
      少年问:"这是哪里?"                    (L=8, 对话)
  改: 老人将瓷碗放在桌上，目光在少年脸上停留了几秒，然后收回视线，声音不高不低:"青云宗。"(L=36, 动作+叙述+对话)
      少年愣住了。                                                           (L=5, 揭示)
      青云宗？                                                               (L=4, 疑问揭示)
      那个传说中的修仙门派？                                                   (L=8, 疑问揭示)
  Δ: 把3句短句对话 → 1句长句（含动作+叙述+对话）+ 3句极短揭示（产生节奏）

操作4: 抽象情绪 → 具体动作
  原: 他很紧张。                           (L=4, 抽象)
  改: 他的手指在刀柄上停了半秒，然后继续往前走去。 (L=20, 具体动作)
  原理: 让读者从动作推断情绪，不直接告诉读者"他很紧张"
```

## 八、为什么这样写？（理论依据）

**AI的核心缺陷**：AI 生成文本时倾向于最小化"输出不确定性"，即写"最安全"的句子。
- 最短的句子（L=5-15）→ 语法风险最低
- 最常见的词（"缓缓地""淡淡地"）→ 训练数据中出现频率最高
- 陈述句（句号）→ 问号/叹号意味着需要读者反馈，AI倾向避免
- 均匀段落长度 → 减少结构性决策的不确定性

**真人的写作策略**：真人作者最大化"读者的情绪波动"，具体手段是:
- 句长变异（CV_L 大）→ 读者注意力被节奏吸引
- 长短交替（下降模式）→ 在读者心理制造"释放感"
- 极短揭示句（p_reveal ≈ 0.10）→ 制造悬念钩子
- 动作替代情绪词 → 让读者"脑补"情绪，产生代入感

**数学表达式**：
```
AI文本: P(word) ≈ freq(word|train_data), 高概率词堆叠
真人文本: P(word) ≈ freq(word|genre) × P(emotion_i|word) × P(rhythm_k|position)
        = 题材语言风格 × 情绪曲线 × 位置依赖的节奏
```

你的任务是: **在每一段写作中，让参数更接近真人分布，远离AI的最小风险分布。**
