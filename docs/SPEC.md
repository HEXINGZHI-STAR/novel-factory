## 1. 产品概述

### 1.1 产品定位

**盘古**是一个面向网络小说创作者的企业级AI辅助写作系统，核心定位是：
- **智能创作引擎**：通过五车间流水线（RAG知识检索 → 设定预处理 → 初稿生成 → 逻辑质检 → 文笔精修）实现高质量章节生成
- **长篇一致性保障**：通过伏笔追踪、角色状态管理、世界观词条(Lorebook)强制注入等机制，确保百万字长篇的叙事一致性
- **平台适配专家**：内置番茄、七猫、起点等主流平台的专属写作规则，确保生成内容符合平台调性

### 1.2 目标用户

| 用户类型 | 需求痛点 | 盘古解决方案 |
|---------|---------|-------------|
| 个人作者 | 效率低、质量不稳定 | 五车间流水线 + Prompt增强 |
| 小型工作室 | 多项目管理、批量生产 | 项目管理 + 批量生成 |
| 资深写手 | 长篇一致性、伏笔管理 | 状态追踪 + Lorebook |

### 1.3 核心价值主张

> **"让AI成为你的创作副驾驶，而非替代者"**

盘古不追求"一键生成爆款"，而是提供：
1. **质量可控**：五车间流水线确保每个环节都有质检
2. **风格可调**：40+创作模式、8套平台规则，满足不同题材需求
3. **长篇可靠**：伏笔追踪、角色状态、世界观约束，确保百万字不出戏

### 2.2 五车间流水线

| 车间 | 名称 | 核心职责 | 输出 |
|------|------|---------|------|
| W0 | 主旨锚定 | 锁定核心主旨，阻断"万能主旨" | 一句话主旨 + 情绪锚点 + 反直觉成分 |
| W1 | 设定预处理 | 从冷库提取相关设定，生成热库 | 本章热库（人物状态、物品、场景约束） |
| W2 | 正文初稿 | 按热库约束生成初稿 | 2000字初稿 |
| W3 | 逻辑质检 | 检测OOC、逻辑漏洞、钩子强度 | 质检报告 + 修正方案 |
| W4 | 文笔精修 | 氛围技法、感官描写、文笔润色 | 精修成品 |

**关键设计**：W0是阻断型闸门，不通过则不进入后续车间，确保"方向正确"。

## 3. 功能模块

### 3.1 创作模式系统

**当前已支持40+创作模式**，按题材分类：

| 类别 | 模式示例 | 核心特点 |
|------|---------|---------|
| 都市异能 | urban_power, crazy_lit | 爽点密集、节奏快、金手指展示 |
| 历史穿越 | history_scholar, historical | 考据严谨、权谋博弈 |
| 治愈生活 | healing_life_v2 | 情绪释放、半沢式蓄力、日剧质感 |
| 规则怪谈 | rule_mystery | 设定即规则、禁止解释 |
| 女频Solo | female_solo | 无CP大女主、事业线优先 |
| 悬疑民俗 | folk_horror | 中式恐怖、氛围营造 |

**模式配置文件结构** (`modes/{mode}.json`)：
```json
{
  "mode_id": "healing_life_v2",
  "name": "治愈生活流V2",
  "core_principle": "情绪释放型叙事...",
  "w2_special": {
    "dialogue_priority": "日常对话+食物触发...",
    "hook_types": ["日常微澜", "善意崩溃"],
    "forbidden_hooks": ["悬念收尾"]
  },
  "w4_special": {
    "sensory_priority": ["触觉", "听觉", "视觉"],
    "atmosphere_techniques": ["坂元式物件运用", "日剧式无用细节"]
  },
  "chapter_structure": {...},
  "success_metrics": {...}
}
```

### 3.2 平台适配系统

**当前已支持8+平台**，每个平台有完整的写作规则：

| 平台 | 核心逻辑 | 句长限制 | 对话率 | AI痕迹词 |
|------|---------|---------|--------|---------|
| 番茄小说 | 追读留存 | ≤25字 | ≥40% | 100+禁用词 |
| 七猫小说 | 爽点密度 | ≤30字 | ≥35% | 80+禁用词 |
| 起点中文 | 长期订阅 | ≤35字 | ≥30% | 60+禁用词 |

### 3.3 RAG知识检索

**架构**：HNSW向量索引 + 语义检索

**检索内容**：
- 775本参考书籍分析
- 739条写作技法
- 37种题材模板
- 用户项目专属素材

**检索策略**：按车间类型（w1/w2/w3/w4）选择不同检索向量，确保上下文相关性。

### 3.4 可观测性系统

**8项治愈系评分指标**：

| 指标 | 权重 | 检测内容 |
|------|------|---------|
| 情绪释放密度 | 15% | 释放点分布合理性 |
| 对话率 | 15% | 对话占比是否达标 |
| 句长分布 | 15% | 平均句长、长短句交替 |
| 感官描写 | 10% | 触觉/听觉/视觉细节 |
| AI痕迹词 | 15% | 禁用词出现频率 |
| 钩子强度 | 15% | 章末钩子质量 |
| 角色一致性 | 10% | OOC检测 |
| 剧情连贯 | 5% | 与前文衔接 |

**其他检测器**：
- 情绪曲线检测（独立于LLM的确定性算法）
- 热血浓度检测（JUMP五段结构）
- 人物弧光检测（英雄旅程8阶段）
- 叙事张力曲线

**统一数据库** (`unified_novel.db`)：

## 5. 接口规范

### 5.1 REST API

**基础URL**: `http://127.0.0.1:5001/api/v7`

| 接口 | 方法 | 功能 |
|------|------|------|
| `/health` | GET | 系统健康检查 |
| `/generate` | POST | 执行五车间流水线 |
| `/write` | POST | 一体化写章节（含状态感知） |
| `/anchor` | POST | W0主旨锚定 |
| `/fusion/check` | POST | 模式混搭兼容性检查 |
| `/observability/score` | POST | 文本质量评分 |
| `/observability/curve` | POST | 情绪曲线检测 |
| `/rag/search` | POST | RAG知识检索 |
| `/library/validate/<project>` | GET | 项目三库验证 |

### 5.2 Qclaw集成方案

**集成方式**：Skill插件

```json
{
  "name": "pangu-novel-writer",
  "description": "盘古AI小说创作助手",
  "commands": [
    {"pattern": "写第{n}章", "action": "call_api", "endpoint": "/write"},
    {"pattern": "检查质量", "action": "call_api", "endpoint": "/observability/score"},
    {"pattern": "搜索技法", "action": "call_api", "endpoint": "/rag/search"}
```

## 9. 附录

### A. 术语表

| 术语 | 定义 |
|------|------|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| HNSW | Hierarchical Navigable Small World，向量索引算法 |
| Lorebook | 事前强制注入的世界观词条（受NovelAI启发） |
| Beat Sheet | 章节级故事节拍约束（受Sudowrite启发） |
| W0-W4 | 五车间流水线：锚定→预处理→初稿→质检→精修 |

### B. 技术栈

- **语言**: Python 3.10+
- **框架**: Flask (API), Loguru (日志)
- **向量检索**: FAISS / HNSWLib
- **LLM路由**: LiteLLM
- **数据库**: SQLite (unified_novel.db)
- **前端**: 待定（React/Vue）

- 盘古V7.5后端代码: `backend/app_v7.py`
