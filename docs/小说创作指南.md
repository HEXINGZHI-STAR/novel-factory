## 系统简介

盘古AI写作系统是一个基于大语言模型的小说创作辅助工具，主要特点：
- 1,394本参考小说库
- 12种创作模式
- 四大平台适配（番茄、起点、七猫、晋江）
- 交互式操作界面

## 创作流程

### 1. 创建项目

#### 步骤1：确定基本信息
- **书名**：简短有力，有吸引力
- **类型**：短篇（3-10章）、中篇（10-30章）、长篇（30章以上）
- **平台**：选择目标平台
- **模式**：选择创作模式
- **一句话简介**：用一句话概括核心设定

#### 步骤2：创建项目结构
```
项目名/
├── state.json          # 项目状态文件
├── 大纲/
│   ├── 总大纲.md      # 完整大纲
│   └── 章节规划.md    # 每章规划
├── 正文/              # 章节文件
└── 设定集/            # 人物、世界观设定
```

---

### 2. 写大纲

#### 大纲必须包含的内容：
```markdown
# 书名 总大纲

## 一句话核心概念
（用一句话概括小说核心设定）

## 核心卖点
1. （卖点1）
2. （卖点2）
3. （卖点3）

## 主要人物
### 主角
- 身份：
- 性格：
- 核心目标：
- 核心创伤/执念：

### 关键配角
- 角色1：（简要描述）
- 角色2：（简要描述）

## 章节规划
1. 第1章：（本章任务）
2. 第2章：（本章任务）
...
```

---

### 3. 写章节

#### 每章2000字左右，七猫风格要点：
1. **开篇钩子**：前300字必须抓住读者
2. **节奏快**：每章有小高潮
3. **爽点密集**：每章至少有1个爽点
4. **结尾留悬念**：让读者想看下一章
5. **对话自然**：符合人物性格

## 七猫风格要点

### 1. 开篇钩子
七猫小说开篇必须立即抓住读者，可以用以下方式：
- 主角身处绝境
- 获得金手指/系统
- 发现惊天秘密
- 重生/穿越

#### 好的开篇示例：
```
“林辰！你又在上课睡觉！”
一声怒喝把林辰从美梦中惊醒……
（第1章开头）
```

---

### 2. 节奏控制
- **短篇**：每章1个大冲突
- **中篇**：每3章1个小高潮
- **长篇**：每10章1个大高潮

---

### 3. 爽点设计
每章必须有爽点，常见爽点类型：
- 打脸反派
- 获得新能力
- 英雄救美
- 解开悬念
- 扮猪吃老虎

---

### 4. 章节结尾
每章结尾必须留钩子：
- 突然出现新敌人
- 发现惊人秘密
- 主角陷入危机
- 情感大转折

## 常见问题

### Q: AI输出格式不对怎么办？
A: 在提示词结尾强调：请直接输出正文，不要其他内容！

### Q: 前后设定矛盾怎么办？
A: 写新章节时，把前面章节的关键信息放在提示词开头。

### Q: 字数不够怎么办？
A: 增加细节描写，增加对话，增加小情节。

## 创作建议

1. **先写大纲，再写正文**：避免后期卡文
2. **每章留钩子**：让读者想继续看
3. **保持风格统一**：同一本小说风格要一致
4. **人物性格一致**：人物言行要符合设定
5. **定期检查设定**：避免写崩
