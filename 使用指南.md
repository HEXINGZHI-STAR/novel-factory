# 盘古AI 使用指南 v2.0

## 一、快速开始

### 1. 配置 API Key

编辑 `.env` 文件：
```bash
DEEPSEEK_API_KEY=sk-你的key
```

### 2. 三个最常用命令

```bash
# 列出所有项目
python pangu_workshop.py projects

# 查看项目状态
python pangu_workshop.py status -p "开门见尸"

# 写一章
python pangu_workshop.py write -p "开门见尸" -c 2
```

## 二、写章（核心功能）

### 命令

```bash
python pangu_workshop.py write -p "项目名" -c 章号 [选项]
```

### 模式开关

| 模式 | 命令 | 车间 | 适用场景 |
|------|------|------|------|
| **工坊（默认）** | `--workshop` | W0→W1→W2→W3→W4→W5 | 日常写作，质量优先 |
| **快速** | `--fast` | W0→W2→W4 | 赶进度，跳过质检和导出 |

### 五车间说明

| 车间 | 做什么 | 调用AI |
|------|------|------|
| W0 主旨锚定 | 读取 state.json，提取角色/伏笔/设定 | ❌ 纯逻辑 |
| W1 设置检查 | WriteGates 关卡 + 构建章节热库 | ❌ 纯逻辑 |
| W2 骨架初稿 | PromptBuilder 17层注入 → DeepSeek 生成 | ✅ 21-30s |
| W3 逻辑质检 | 正则检查 AI味词汇/句长/对话率 | ❌ 纯逻辑 |
| W4 精修定稿 | 带 QC 反馈 → DeepSeek 润色 | ✅ 22-35s |
| W5 导出收尾 | 写文件 + 更新 state + 五路投影 + 情报 | ❌ 纯逻辑 |

### 示例

```bash
# 工坊模式写第3章（默认）
python pangu_workshop.py write -p "开门见尸" -c 3

# 快速模式写第3章
python pangu_workshop.py write -p "开门见尸" -c 3 --fast

# 指定章任务
python pangu_workshop.py write -p "开门见尸" -c 3 -t "沈让来找姜渺——上一个能看到预知的中介已经死了"

# 仅验证不调AI
python pangu_workshop.py write -p "开门见尸" -c 3 --dry-run
```

### 批量写章

```bash
# 连续写第3到第8章，每章间隔3秒
python pangu_workshop.py batch -p "开门见尸" --from 3 --to 8 --delay 3
```

## 三、审查与分析

### 审查章节

```bash
python pangu_workshop.py review -p "开门见尸" -c 1
```

输出：7维审查报告（设定/时间线/叙事/角色/逻辑/技法/读者体验）+ AI风险 + 质量后验 + 建议。

### 智能诊断

```bash
python pangu_workshop_smart.py diagnose -p "开门见尸"
```

输出：全维度诊断（审查+伏笔网络+进度预测+经济学+下章策略+推荐任务）。

### 文本分析

```bash
# 对任意文本跑数学分析
python mcp_server.py pangu_analyze text="你的文本内容"
```

## 七、紧急情况处理

| 问题 | 解决 |
|------|------|
| API Key 失效 | 去 platform.deepseek.com 重新生成，更新 `.env` |
| 写章失败 | 看错误日志，API 问题换 Key，其他问题降低到 `--fast` |
| 句均太短 | 切回 `--workshop` 模式，W3 会自动检测 |
| 质量偏低 | `pangu_workshop_smart.py diagnose` 看诊断建议 |
| Claude 绕过系统 | 说"你没有调 Pipeline，重来" |
