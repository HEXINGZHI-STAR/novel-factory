# 写作压力测试

## 目标

在写章之前，从读者/编辑/对手作者三个角度质疑当前章的设计，找出薄弱点。

## 质疑维度

### 1. 设定质疑
```
□ 这个设定是否与已锁定规则矛盾？
□ 读者会不会觉得"太方便了"？（金手指/巧合/天降）
□ 如果删除这个设定，剧情还能成立吗？
```

### 2. 伏笔质疑
```
□ 这条伏笔埋设章和回收章之间，读者会不会忘？
□ 有多少读者能提前猜到回收方式？（<30%=成功，>70%=失败）
□ 如果只用一句话回收这条伏笔，那句话是什么？
```

### 3. 逻辑质疑
```
□ 角色的每个行为都有动机吗？（不能用"剧情需要"解释）
□ 信息揭示的时机——角色知道这件事的时间点合理吗？
□ 如果去掉这一章，对整本书的影响有多大？
```

### 4. 读者体验质疑
```
□ 7秒扫描测试：读者快速翻过这章，能看到几个"想停下来读"的点？
□ 情绪曲线：连续紧张超过3章了吗？连续平淡超过2章了吗？
□ 这章结束后，读者最可能做什么——翻页？放下？弃书？
```

## 执行流程

### Step 1：加载项目上下文

```bash
python pangu_workshop.py status -p {project}
python pangu_workshop.py validate -p {project}
```

### Step 2：提取本章规划

```
读:
  {project}/大纲/总纲.md → 找到第N章的规划
  {project}/state.json → 查看伏笔表/设定锁
  {project}/正文/第{N-1}章.txt → 上一章结尾钩子
```

### Step 3：逐维质询

Claude 对每个质疑维度逐一回答——必须是"会出问题"的具体风险，不是"可能有问题"的泛泛而谈。每项给出风险等级（🔴阻断 / 🟡警告 / 🟢通过）。

### Step 4：输出风险报告

```markdown
# 第N章 压力测试报告

## 阻断风险
### 设定矛盾
- 位置: 第N章第X段 → 前文规则: Ch5 locked_rules
- 问题: ...
- 建议: ...

## 警告项
...

## 通过项
...
```
