# 盘古质量审查

## 核心理念

**机器做规则检查，Claude 做审美判断。**

W3 的规则检查（AI味词汇、句长、对话率）机器已经做了。
本技能是 Claude 的深度审查——需要理解力、审美力、共情力的环节。

## 审查维度

### 1. 设定一致性（权重：阻断）

```
检查项:
□ 角色能力是否与设定卡一致？（不能突然多了没写过的能力）
□ 世界观规则是否被违反？（硬约束不可破）
□ 前文已锁定的设定是否被矛盾覆盖？
□ 新出现的设定是否与已有设定兼容？
```

比对源：`设定集/世界观.md` + `设定集/主角卡.md` + state.json 的 `locked_rules`

### 2. 时间线（权重：阻断）

```
检查项:
□ 本章时间点与前章是否连续/合理跳跃？
□ 事件发生顺序是否可能（不会同一时间做两件互斥的事）？
□ 如果用了倒叙/插叙，读者能否分辨？
□ 伏笔的时间标注是否正确（埋设章/回收章）？
```

### 3. 叙事连贯（权重：高）

```
检查项:
□ 上一章的结尾钩子本章接住了吗？
□ 场景转换是否顺畅（不会跳得太快/太慢）？
□ 对话内容是否与说话人的已知信息一致？（不能说出他不知道的事）
□ 人物关系是否连续？（不能上一章刚认识这章就生死之交）
```

### 4. 角色一致性（权重：高）

```
检查项:
□ 主角行为是否在"OOC警戒"范围内？
□ 核心性格是否稳定？（可以成长，不能突变）
□ 配角是否有独立动机？（不只是为主角服务）
□ 对话语气与角色设定是否匹配？
□ 情绪反应是否合理（不会过度或不足）？
```

### 5. 逻辑自洽（权重：中）

```
检查项:
□ 情节因果是否成立？（A导致B，不能凭空出现）
□ 信息揭示是否合理？（角色知道的信息有获取途径）
□ 没有"第7章已经知道的事在第12章当新发现"的矛盾
□ 数字/金额/时间等具体数值前后一致
```

### 6. 技法达标（权重：中）

根据盘古核心技法栈检查：
```
□ 句均汉字：12-18字（治愈系）/ 按项目调整
□ 短段比：50-70%
□ 触觉优先级：30-37%感官描写必须包含触觉
□ 物件叙事：12-15处/千字的象征性物件
□ 情绪释放节点：本章是否贡献了情绪递进
□ 金句密度：1-2句可截图传播的金句
```

### 7. 读者体验（权重：低，但值得注意）

```
□ 本章读完后的情绪余韵是什么？
□ 结尾是否让人想立刻翻下一章？
□ 有没有"不明觉厉但细想不通"的地方？
□ 节奏是否单调（连续紧张/连续平淡）？
```

## 执行流程

### Step 1：加载审查上下文

```
必读:
  {project}/正文/第{chapter_num}章.txt    → 审查对象
  {project}/设定集/主角卡.md               → 角色基准
  {project}/大纲/总纲.md                   → 伏笔表+总方向
  {project}/.webnovel/state.json           → locked_rules + 角色状态

按需:
  {project}/正文/第{chapter_num-1}章.txt   → 衔接检查
  {project}/设定集/世界观.md               → 设定约束
```

### Step 2：逐维审查

对每个维度下的每个检查项，Claude 逐条给出判断：
- ✓ 通过（可附简要理由）
- ⚠ 警告（不阻断但建议修改，必须引用正文具体段落）
- ✗ 阻断（必须修改，引用正文+引用约束源，附修改建议）

### Step 3：生成审查报告

写入 `{project}/审查报告/第{chapter_num}章-审查报告.md`：

```markdown
# 第{chapter_num}章 审查报告

## 总评
- 总体评分: {score}/100
- 阻断项: {blocking_count}
- 警告项: {warning_count}
- 判定: [通过 / 有条件通过 / 需要返工]

## 分维评分
| 维度 | 评分 | 判定 |
|------|------|------|
| 设定一致性 | /100 | |
| 时间线 | /100 | |
| 叙事连贯 | /100 | |
| 角色一致性 | /100 | |
| 逻辑自洽 | /100 | |
| 技法达标 | /100 | |
| 读者体验 | /100 | |

## 阻断项（必须修改）
### {标题}
- 位置: {正文引用}
- 冲突约束: {约束源引用}
- 修改建议: {具体方案}

## 警告项（建议修改）
### {标题}
- 位置: {正文引用}
- 问题: {描述}
- 建议: {具体方案}

## 亮点
- {至少1条，值得保留或强化的地方}
```

### Step 4：用户裁决

如有阻断项：
- 列出所有阻断项，等用户决定：立即返工 / 记录不阻断 / 标记为已知问题
- 用户裁决后，将决定写入审查报告的"裁决记录"段

### Step 5：写回状态

```bash
cd {PANGU_ROOT}
python -c "
import json
from pathlib import Path
# 更新 state.json 的 review_checkpoints
state_path = Path('{project}/.webnovel/state.json')
state = json.loads(state_path.read_text(encoding='utf-8'))
state.setdefault('review_checkpoints', []).append({
    'chapter': {chapter_num},
    'score': {score},
    'passed': {passed},
    'reviewed_at': '{timestamp}'
})
state_path.write_text(json.dumps(state, ensure_ascii=False, indent=2), encoding='utf-8')
print('Review checkpoint saved.')
"
```

## 阻断判定标准

阻断级 issue (blocking=true) 必须满足：
- 设定硬约束被违反（不可逆的规则矛盾）
- 时间线出现不可能的先后顺序
- 主角出现严重 OOC（与性格底线矛盾）
- 前文已回收的伏笔被当新线索重新"发现"

非阻断标记为 `blocking=false`，记录建议即可。
