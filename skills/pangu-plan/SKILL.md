# 盘古规划流程

## 核心理念

**Claude 做创作判断，不委托 subagent。** 大纲是创作行为，不是工程任务。

## 模式

| 模式 | 说明 |
|------|------|
| 默认 | 规划当前卷（基于总纲的卷划分） |
| `--full` | 全本规划：总纲→每卷卷纲→时间线→章纲 |
| `--volume N` | 只规划第 N 卷 |

## 硬规则

- Claude 必须亲自读总纲，不得委托 subagent
- 章纲不可与已写正文矛盾
- 伏笔必须标注埋设章和回收章
- 设定增量写入，不覆盖已有设定

## 执行流程

### Step 1：读取基础约束

```bash
必读文件:
  {project}/大纲/总纲.md
  {project}/.webnovel/state.json
  {project}/设定集/世界观.md
  {project}/设定集/主角卡.md

已写正文:
  ls {project}/正文/   → 确认已写到哪章
```

### Step 2：生成/更新规划

Claude 直接生成以下文件：

**卷纲** (`大纲/第{N}卷-卷纲.md`):
```
- 卷名
- 章节范围
- 核心冲突
- 卷末高潮（一句话描述钩子）
- 主角本卷成长节点
```

**时间线** (`大纲/第{N}卷-时间线.md`):
```
| 章节 | 时间点 | 关键事件 | 伏笔操作 | 人物出场 |
|------|--------|----------|----------|----------|
```

**章纲** (`大纲/第{N}卷-详细大纲.md`):
```
每章包含:
- 章目标（本章完成什么）
- 开头钩子
- 核心场景（1-3个）
- 结尾钩子（悬念/情绪/信息差）
- 本章伏笔操作（埋设/推进/回收）
- 本章禁区
```

### Step 3：增量写入设定集

规划过程中发现的新设定，增量写入：
- 新角色 → 更新 `设定集/主角卡.md` 或新角色卡
- 新地点 → 写入 `设定集/世界观.md` 地点清单
- 新规则 → 写入世界观核心规则区
- 新伏笔 → 确保总纲伏笔表包含

### Step 4：更新约束文件

```bash
# 更新 state.json 的卷规划
cd {PANGU_ROOT}
python -c "
from pangu_core.state_sync import StateSync
from pangu_core.db import get_db
# 增量同步规划数据
print('Plan synced to state.')
"
```

### 质量自检

完成前 Claude 自问：
- □ 每章都有明确的"结尾钩子"吗？
- □ 伏笔表的埋设/回收章节号自洽吗？
- □ 角色成长曲线连续吗（不会突然变强/变性格）？
- □ 高潮章节之间有"呼吸章"吗（连续高强度=读者疲劳）？
- □ 最后一章能回收主线伏笔吗？
